# SOC 2 Type 1 vs. Type 2 for a B2B SaaS Startup

## What is SOC 2?

SOC 2 is a security audit framework developed by the AICPA that's widely used in B2B software. It evaluates whether a company has proper controls around security, availability, and data handling. An independent auditor reviews your controls and issues a report that you can share with customers as proof of your security posture.

---

## Type 1 vs. Type 2: What's the Difference?

**SOC 2 Type 1** is a point-in-time assessment. The auditor looks at your controls on a specific date and confirms that they are designed correctly. Think of it as a photograph: "On this day, the controls look right."

**SOC 2 Type 2** evaluates whether your controls not only exist but have been working effectively over a period of time — typically 6 to 12 months. Think of it as a video: "Over this period, we watched the controls operate and they consistently worked."

The key differences:

| | Type 1 | Type 2 |
|---|---|---|
| Scope | Design effectiveness only | Design + operating effectiveness |
| Time period | Single date | 6–12 months |
| Effort | Less | Significantly more |
| Time to complete | 2–4 months | 9–18 months |
| Customer trust value | Moderate | High |
| Cost | Lower | Higher |

---

## Which Should You Pursue?

For an enterprise customer asking for a SOC 2 report, **they almost certainly want Type 2**. Type 1 is sometimes accepted as a stepping stone, but enterprise procurement and security teams know that Type 1 only proves your controls are designed — not that they actually work day to day.

That said, if you have no SOC 2 work done yet, going straight to a Type 2 requires patience because:
- You need to implement controls first
- Then you need an observation period of at least 6 months
- Then the auditor does fieldwork and issues the report

**Recommended approach for your situation:**

1. Start implementing controls immediately — the clock doesn't start until controls are actually running
2. Consider getting a Type 1 report in 3–4 months to show the enterprise customer you're serious and making progress
3. Use the Type 1 as a bridge while your 6-month observation period runs
4. Complete the Type 2 roughly 12–14 months from today

This "Type 1 bridge" approach is very common for first-timers with an impatient enterprise customer.

## Typical Timeline

### Getting a Type 1 (fastest path to something)

- **Months 1–2:** Implement core security controls, write policies, conduct a risk assessment
- **Months 2–3:** Readiness review, close gaps, gather evidence
- **Months 3–4:** Auditor fieldwork and report issuance
- **Total: 3–5 months**

### Getting a Type 2 (what enterprise customers really want)

- **Months 1–2:** Implement controls and policies (same as Type 1 prep)
- **Months 2–8:** Observation period — controls must operate consistently with evidence
- **Months 8–10:** Auditor fieldwork (they test samples across the period)
- **Months 10–12:** Report issued
- **Total: 10–14 months from scratch**

---

## Key Controls to Build Now

Regardless of Type 1 or 2, you'll need:

- **Access controls:** Multi-factor authentication, least privilege access, user provisioning and deprovisioning procedures, periodic access reviews
- **Incident response:** A documented plan that gets tested or rehearsed
- **Change management:** A process for reviewing and approving code and infrastructure changes
- **Risk assessment:** An annual documented process for identifying security risks
- **Vendor management:** Inventory of critical vendors and reviews of their security
- **Monitoring:** Logging and alerting on your systems for anomalies
- **Policies:** Written information security policy, acceptable use policy, and others

---

## Practical Next Steps

1. **Talk to your enterprise customer** — ask what they'll accept (Type 1 now, Type 2 later?) and what scope they need (security only, or also availability?)
2. **Engage an auditor early** — only a licensed CPA firm can issue a real SOC 2 report; good firms book up months in advance
3. **Consider a compliance platform** — tools like Vanta, Drata, or Secureframe can automate evidence collection and speed up readiness significantly for a startup
4. **Start with security basics** — MFA everywhere, access reviews, documented policies, logging

The good news: as a cloud-native startup, many of the controls you need are achievable quickly if you use the right tools and commit the time.
